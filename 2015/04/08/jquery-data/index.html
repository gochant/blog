JavaScript 基础：jQuery 数据存储

发表于 2015-04-08   |   分类于 Frontend

jQuery 的数据存储一开始主要为了解耦事件与DOM之间的直接联系引入的，后来演变为动画、事件队列等都用到数据存储系统，是jQuery的核心构建之一。

用法

有两种方法：

  • 静态方法 $.data 底层方法，不建议直接使用
  • 实例方法 $('selector').data √ 推荐

两种方式的区别

  1. jQuery.data()可以实现为dom元素或js对象添加缓存
  2. $('selector').data()是对前者的扩展，其目的是可以方便的通过选择器为 多个 dom元素添加缓存数据

注意

  • 由于浏览器与插件和外部代码的交互方式， data 方法不能用在 <object>（除非它是Flash插件）、<applet>、<embed> 元素上。（由于jQuery 2.x不支持IE8-，所以在这个版本就没有这个限制了）
  • .removeData() 仅会删除来自jQuery内部.data()缓存中的数据， 并且元素上任何相应的data-属性不会被删除。后调用data()，会重新检索data-的属性的值。 为了防止这种情况，可以在.removeData()旁边使用.removeAttr()来移除data-属性（jQuery1.4.3以前，data()没有使用data-属性，所以不存在这类问题）。
  • .data 方法目前并不提供在XML文档上跨平台设置数据，作为Internet Explorer不允许通过自定义属性附加数据。
  • undefined不是认可的数据值。比如这样调用.data( “name”, undefined ) ，将返回 “name”对应的数据， 也就是等同于调用.data( “name” )。

jQuery 源码解读

（以下无特别说明基于 jQuery 2.1.3 源码解析）

为某个对象附加数据，最简单的方式就是在这个对象上添加属性存储，由于考虑到DOM对象的一些特殊性（内存泄露风险和IE bug），在DOM上添加复杂的属性值是一个不太好的方式。

jQuery的做法是为每个要附加自定义数据的对象添加一个自定义属性，这个属性只存放访问实际数据的一个key，而实际的数据放在另一个与该对象毫无关系的一个公共对象中。

自定义属性的命名

该属性名形式为：jQueryxxxxxxxxx，值为一个累加的数字 uid。这个值是一个key，通过它访问存储对象中的对应数据。

属性名由 jQuery.expando + uid（Data.uid） 生成，expando是一个为了尽量避免与用户命名名称相冲突的一个随机字符串

数据的存储方式

数据存储的地方就是一个普通的JS空对象中，通过一个映射关系与直接的DOM元素发生关联，在存储结构上实现上，jQuery 1.x 和 jQuery 2.x 有区别：

  • jQuery 1.x

数据存储在可外部访问的数据存储对象$.cache中，为了避免jQuery内部使用的数据和用户自定义的数据发生冲突，内部数据直接存储在该对象cache上，自定义数据存储在cache.data上；

对于DOM元素和普通JS对象存储方式也有区别：DOM对象的数据存储在jQuery.cache中，普通JS对象的数据存储在该对象中。

  • jQuery 2.x

存储对象有两个，并且这些对象都是 Data 类的实例，外部不能访问。

并且，所有元素或对象的存储方式都是存在缓存对象中，不存在本身对象中。
