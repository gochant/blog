也谈 JavaScript 隐式类型转换

发表于 2016-05-24   |   分类于 Frontend

JS 隐式类型转换是一个坑，平时在开发过程中有时会碰到，例如 [0] == false，1 + '0' 之类的，这里就简单说一下比较容易搞混的几个点

等值判断（==）

我遵循的准则是在开发过程中都用 ===，禁止使用 == 和 !=（只在 判定 val == null 时使用），这样做能够规避许多因不清楚类型转换引起的bug，但有时我们使用第三方库可能也会用到该操作符，因此也有必要有所了解

关于 == 的操作步骤定义在规范 equlity operators 的 runtime semantics evaluation
这一小节的第一段，主要关注的是最后一步，对左值右值进行Abstract Equality Comparison，这里就描述了 == 是怎么判断的，摘抄如下：

  1. If Type(x) is the same as Type(y), then
    1. Return the result of performing Strict Equality Comparison x === y.
  2. If x is null and y is undefined, return true.
  3. If x is undefined and y is null, return true.
  4. If Type(x) is Number and Type(y) is String, return the result of the comparison x == ToNumber(y).
  5. If Type(x) is String and Type(y) is Number, return the result of the comparison ToNumber(x) == y.
  6. If Type(x) is Boolean, return the result of the comparison ToNumber(x) == y.
  7. If Type(y) is Boolean, return the result of the comparison x == ToNumber(y).
  8. If Type(x) is either String, Number, or Symbol and Type(y) is Object, return the result of the comparison x == ToPrimitive(y).
  9. If Type(x) is Object and Type(y) is either String, Number, or Symbol, return the result of the comparison ToPrimitive(x) == y.
  10. Return false.

总结一下大概可归结为：如果不是 Object，则 ToNumber 进行比较，如果是 Object，则 ToPrimitive 进行比较，总之其实都是往 number 类型上进行靠拢来比较

关于 ToPrimtive：该方法签名为 ToPrimitive ( input [ , PreferredType ] )，在未传入第二个参数时，最终采用 number 作为参数，这个与 string 作为参数的最大区别是，number 先调用 valueOf，再调用 toString，而 string 正好相反，在 MDN 这篇文章中谈到 toPrimitive 时，我觉得文中说法可能有误

一些例子

  1. undefined == null true
  2. '\n 123 \t' == 123 true
  3. [0] == false true

加减法操作（+，-）

这个定义在规范 The Addition Operator 这一节，主要注意一点就是如果左值或右值至少有一个为 string（如果是对象，则 ToPrimitive 后，也为 string），那么调用 ToString，否则都调用 ToNumber

举个例子

注意以上例子 {}+1 如果直接控制台执行，会得到 1，这是因为解析时将 ‘{}’ 解析为块语法，所以就变成 +1，同理 {}+'1'也是一样，这点要注意

当数字操作时，有几个感觉比较特殊的：

  1. 任何与 NaN 相加都为 NaN
  2. 同等方向 Infinity 操作，为 +-Infinity
  3. 不同方向 Infinity 加，为 NaN
  4. 两个绝对值相等值，不同符号相加，为 +0
  5. 仅两个 -0 相加，为 -0

减法操作都转换成数字，这个就不做过多赘述
