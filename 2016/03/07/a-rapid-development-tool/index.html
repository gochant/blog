一个快速开发工具

发表于 2016-03-07   |   分类于 Architecture

CMMI 体系和严谨的软件工程讲究整个开发生命周期的所有活动都要留下相关证据，并且整个路径的相关记录都要保持一致性。举个例子：如果发生变更，那么相应会有变更的申请单以及评审表，某个文档发生变更，其他影响到的文档或代码都应该变更并形成相应记录，并且维持好对应的需求跟踪矩阵。

最基本的，其实就是需求-设计-编码-测试这些关键子过程要保持一致，在项目开发过程中，经常由于资源、环境配置、管理不到位，造成以下几个方面的问题：

  • 需求详细程度不够
  • 工期太短，无法形成有效设计
  • 开发人员水平参差不齐，编码质量无法维持一致性，增加重构成本
  • 后期变更有些过程文档没有照顾到，无法进行双向追溯

这里面的核心矛盾就是开发速度与成果质量，如果能够让开发人员不写代码或尽量少些代码，那么就会降低出错的几率，最好是能够从需求直接生成实现，那是再好不过了。

快速生产的一些途径

我见过两种能够显著提高开发速度的工具：

  • 可配置系统
    • 优点：热插拔、可视化配置，用户也可使用，业务逻辑易拷贝
    • 缺点：开发人员不友好（提供一套新的机制，限制较多、扩展一般只能通过插件实现）、性能优化不易（并且可能采用了动态加载，效率会变慢）、深度定制成本增大
  • 代码生成器
    • 优点：仅提供脚手架，即减轻工作量，又高度自由；与现有开发模式一致，易于接受
    • 缺点：开发模式比较原始，不够智能，可能仍有较大开发工作量，效率提高不够显著

从时间和个人偏好以及后期维护角度考虑，我打算做一个可以认为是代码生成器的东西，它面向开发人员，并期望提供一些不一样的东西，主要达到以下几个目标：

  1. 维护一致 维持需求与实现的一致性
  2. 开发效率 最大限度降低开发人员工作量和出错几率
  3. 可扩展 在需要自定义的场景能够任意定制满足其需求

确保一致性

为了实现维护一致性，最好的方式就是一旦改了需求，那么代码就自动更改，单一般来说，需求规格说明书都是Word等非结构化数据，要实现从需求到实现的目的，首先是要将其转换成结构化数据，例如XML，也就是结构化需求（SRS -> SDD -> Code），在我的解决方案，SRS的几个关键部分被结构化如下：

  • 总体功能模块图 -> *.sitemap
  • 功能需求
    • 用户故事 -> *.bmx
    • 数据描述 -> *.edmx
  • 非功能需求（安全、性能……） -> 底层框架（Orzoo）

说明：模块图是一棵树，用站点地图文件可以很好地表达，它会被程序读取用来构建导航和菜单；用户故事这里我创建了一种新的XML数据（业务模型文件，用来描述模型之间的关系，另外用户故事的场景逻辑部分则通过底层框架预置）；数据描述则最直接的方式通过实体数据模型来定义；非功能需求则通过底层框架来保证达到相应的指标

提高开发效率

有了结构化的需求文档，就可以通过程序读取该文档并生成代码，生成代码这里采用了 T4 模板引擎，对于 sitemap 和 edmx 文件，在VS里面都能很方便的维护，对于自创的 bmx 文件，我不得不为它单独写了一个 VisualStudio 的插件去完成可视化的编辑

扩展性

由于代码生成器只是为你生成了你不需要手工编写的代码，因此跟你自己平时开发一样，你可以任意更改和扩展，但出于维持一致性的考虑，开发人员最好不要去动生成器生成的代码，因此对于每一个生成的类，都应该定义成一个 partial class。另外，一种常见的扩展方式是通过AOP的机制注入一些拦截代码，由于时间因素，暂未实现

基础设施

  • 业务模型（bmx）编辑插件：VerGen
  • 轻量级ASP.NET框架：Orzoo
  • 前端框架：veronica（单页面框架）、tinyui（bootstrap扩展库）、veronica-ui（第三方库集合）

Orzoo 组成

  • Orzoo.Core（核心）
  • Orzoo.AspNet（ASP.NET增强）
  • Orzoo.Office（Excel、Word导入导出）
  • Orzoo.WebBase（通用界面的封装）
  • Orzoo.Business（二级缓存、用户行为记录、日志、权限、组织架构、访问控制）

Orzoo 依赖

  • Orzoo.Core (System.Linq.Dynamic)
  • Orzoo.AspNet (Orzoo.Core、AspNet.Mvc、EntityFramework、AutoMapper)
  • Orzoo.Office (NPOI、RazorEngine)
  • Orzoo.WebBase (Orzoo.AspNet、MvcSiteMapProvider)
