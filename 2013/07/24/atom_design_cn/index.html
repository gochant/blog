原子设计

发表于 2013-07-24   |   分类于 Architecture

我们不是在设计页面，我们在设计系统的组件。 ——史蒂芬 · 海

随着web设计工艺的不断演变，我们意识到应该开发一个考虑周到的设计系统，而不是将网页简单的堆砌起来。

很多已经说过创建设计系统这个话题，大多数都是讲应搭建一个基础，例如通用的颜色、排版、网格系统、结构以及其他类似的东西。这些当然很重要，但是我对这些主观上的东西不是很关心，我更感兴趣的是我们的界面组成和我们怎样以一种更有条理的方式构建系统。

在捕捉灵感的过程中，我想到了化学。所有的物质（不论是固体、液体、气体、简单的、复杂的等）是由原子组成的。这些原子组合在一起形成分子，进而组合起来形成更复杂的有机体，最后创造我们宇宙中的所有物质。

类似的，界面由更小的组件构成，意味着我们能将整个界面分割成基本单元，并从这些基本单元开始着手工作。这是原子设计的基本宗旨。

什么是原子设计？

原子设计是创建设计系统的方法。在原子设计中有五种不同的等级：

  1. 原子
  2. 分子
  3. 生物（组织）
  4. 模板
  5. 页面

让我们详细介绍每个阶段。

原子

原子是物质的基本单元，对于与Web界面，原子就是我们的HTML标签，例如 form 标签、input 或 button。

原子也包括更抽象的元素，例如调色板、字体，以及一个界面更无形的方面，例如动画。

原子本质上是相当抽象的，并且它自己本身不是太有用，然而他们作为你模式库中上下文的参考是很有用的，这样你能一目了然的看到你的所有样式。

分子

当我们开始将原子结合在一起时，事情变得更加有趣和有形。分子是一组原子聚合在一起形成的，它是化合物的最小单元。分子呈现他们自身的属性，并作为设计系统的骨干部分进行服务。

举个例子，一个表单标签，输入框和按钮它们本身不是很有用，但是把它们组合起来形成一个表单，它们就能做一些实际的事。

从原子构建分子时建议的准则：只做一件事情并把它做好。分子有可能会很复杂，一般来说它们是一些相对简单的原子的组合和重用。

生物

分子给了我们一些构建模块，我们现在能把它们组合起来形成生物。生物是多组分子结合起来形成的一种界面上相对复杂，独特的部分。

我们开始变得越来越具体化。客户可能不太关心一个设计体系分子层面的东西。但是到了生物层面，我们能看到一些最终界面成形的东西。

Dan Mail（我和一些其他人工作的几个项目）使用 element collages，清晰表达几个关键生物，促进与用户的沟通并塑造出视觉上的方向（所有的这些无需构造完整的对比方案）。

生物可以由相似或者不同的分子类型所构成。例如，一个“报头”生物可能由LOGO、主导航、搜索框、和社交媒体列表等组件构成。但是一个“产品表格”可能由一些相同的分子（可能包含产品图片、名称和价格）一遍一遍重复构成。

从分子构建生物建议的准则：创建独立的、便携的、可重用的组件。

模板

到了模板这个层面，我们脱离化学的领域进入编程语言层面，这样对于客户的交流和最终的输出更有意义。模板主要由成组的生物紧密结合在一些。现在我们能看到很多设计的东西在里面，同时也能看到整个布局的样子。

模板是非常具体的，它为所有这些相对抽象的分子和生物提供上下文环境。模板也能够让用户看到最终设计的位置分布。从我使用这个方法论的经验来看，模板从HTML线框图开始它的生命周期，随着时间的增加不断提高保真度，从而成为最终交付物。在匹兹堡的大胡子工作室采用一套相似的流程，一开始设计是灰度的并且布局很弱，然后在每个位置慢慢增加保真度直到最终设计。

页面

页面是模板的一个实例。在这里，占位内容被替换成真实的内容，给用户最终所见的最准确的描述。

页面是高保真的，因为他们是有形的，它会花费最多的时间，并且会引来人们最多的评论。

页面阶段是必须的，我们能以此来测试设计系统的有效性，并让我们能够不断循环迭代修改我们的分子、生物和模板以满足实际环境的需要。

页面也是测试模板可变性的地方。例如，你可能想表示一个标题包含40个字符时像什么样子，也可能想展示包含340个字符时看起来像什么样子。或者展示一个用户在购物车添加一件商品和十件商品应用折扣时的不同样子。所有的这些特殊实例都会影响我们怎样循环构建我们的系统。

为什么进行原子设计

以上提到的，很多是我们一直就这样做的，即使我们没有以这种特殊的方式有意识的进行思考。

原子设计提供一种清晰的方法论用于制作设计系统。

客户和团队成员能通过实际看到摆在他们前面的这些步骤，更好的解读一个设计系统的理念。

原子设计给我们一种贯穿抽象到具象的能力。有了这些，我们能创建一个提高一致性和可伸缩性，并最终在他们的上下文中显示事物的系统。通过组装而不是解构，我们站在正确之门构建系统而不是事后随意的应用各种模式。

模式实验室

为了将这个方法论应用到我的工作中，我（和伟大的 Dave Olsen 帮助）创建了一个叫做 Pattern Lab 的工具，实际创建这些原子设计系统。我将在后面详细介绍 Pattern Lab，但可随时在Github上迁出它。
