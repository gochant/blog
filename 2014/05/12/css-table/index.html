表格式化

发表于 2014-05-12   |   分类于 Frontend

Table 布局作为规范中单独的章节，相对于 Normal Flow 还是有很多差异，这里简单罗列一下

表显示

元素 ‘display’ 值
table table
tr table-row
tbody table-row-group
thead table-row-group
tfoot table-footer-group
col table-column
colgroup table-column-group
th、td table-cell
caption table-caption

特性

以行为主

显式声明行，列是从单元格行的布局中推导出来的

列

有局限性，只能接受4种样式的设置

  • border：
    只有 border-collapse 为 collapse才能设置列和列祖的边框
  • background：
    只有单元格和行为透明背景时，才可见
  • width
    改属性只是定义了列组的最小宽度
  • visibility
    只有 collapse 值才起作用，设置后该列或列组所有单元格都不显示，设置其他值无效

匿名表对象

如果写HTML的人没有按照标准写html结构，那么css会有一种机制，将这些丢失的作为匿名对象插入，以下是对象插入规则：

  1. 插入匿名row
  2. 插入匿名table

表层级

优先级由上往下依次递减：

单元格、行、行组、列、列组、表

表标题

caption-side：指定标题出现在表格的上面还是下面 （top | bottom）

表单元格边框

有两种边框模型：一种是分隔，一种是合并，通过属性 border-collapse 指定，
包括（collapse 和 separate）

separate 是浏览器的默认方式：border-collapse: separate

  • border-spacing
    设置边框间隔（设置在表格上），

empty-cells

处理空单元格

合并的单元格边框

设置后，会有以下变化：

  1. table的内边距无效
  2. empty-cells 无效
