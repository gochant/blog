JavaScript 基础（一） 语法、表达式、语句

发表于 2014-10-07   |   分类于 Frontend

前言

这一系列文章主要是给团队内的前端新人建立一个较为完整的基础知识体系，每一小节不做太深入的解释，主要是是罗列一些重要的或者平时不会太注意的点

当然，前端老司机也可时常翻看作为巩固基础的检查单，主要是对应 ES3/ES5 规范

词法结构

  • JavaScript 由 UTF-16 编码的 Unicode 字符集编写
  • 区分大小写
  • Unicode 转义序列（为了支持老式机器 \u00e9）
  • 注释
  • 直接量（数字、字符串、正则表达式、null、数组、对象）
  • 标识符
  • 保留字
  • 分号

类型、值、变量

类型分类

六种：

  • 原始类型（primitive type）：数字、字符串、布尔值、null、undefined
  • 对象类型（object type）：普通对象、数组、函数

另一种分类：

  • 可变类型（mutable）：值可以修改的（引用类型）。对象、数组
  • 不可变类型（immutable）：数字、布尔、null、undefined、字符串（均为原始类型）

数字

IEEE 754 64位浮点格式（double）

  • 整型

十进制、十六进制（0x 或 0X）、八进制（0开头，ES规范没有，ES6 严格模式禁止）

  • 浮点型

实数写法
指数写法（1.473E(或e)-32）

算数运算

  • 1/0、1/-0
  • Infinity、-Infinity、NaN
  • NaN与任何值都不相等
  • isNaN、isFinite
  • -0 与 0 全等

二进制浮点数和四舍五入错误： .3-.2 === .2-.1
解决方案：使用小单位，避免小数运算

bug: ES3 中，Infinity 和 NaN 可读可写，ES5中纠正了这个错误

字符串

由一组16位值组成的不可变有序序列

  • 字符串操作均针对 16 位值，不保证合法
  • 长度计算的误区
  • 单引号和双引号
  • 转义字符（\）
  • 字符串的用法
  • 模式匹配

布尔值

  • 真值，假值（null、undefined、0、-0、NaN、””）

null 和 undefined

  • 使用typeof（null：object、undefined：undefined）

bug：ES3中undefined是可读写变量，ES5中修正

全局对象

它的属性包括：全局属性（undefined、Infinity、NaN）、全局函数（isNaN、parseInt、eval）、构造函数（Date、RegExp、String、Object、Array）、全局对象（Math、JSON）

引用：this、window

包装对象

  • 作用于：数字、布尔、字符串
  • 运行机制：即用即创建、用完销毁
  • 显式创建包装对象，区别：typeof、=== 不等

类型转换

  • == 会引发隐式类型转换

显式类型转换

变量

  • 变量作用域，函数作用域，无块级作用域
  • 预编译与 hoisting
  • declarative environment record
  • 作用域链查找，如果找不到，抛 ReferenceError；嵌套函数，每次调用外部函数，内部函数重新定义，因为作用域链不同

表达式与运算符

表达式

表达式是JavaScript的一个短句，JS解释器会将其计算（evaluate）出一个结果

运算符会将简单表达式组合成一个复杂表达式

原始表达式

常量、直接量、关键字、变量

函数定义表达式

属性访问表达式

expression . identifier
expression [ expression ]

  • 先计算前面的表达式，如果是 null 或 undefined，抛出 TypeError
  • 后面如果是 .，则查找标识符指定的属性值；如果是 [，则计算表达式值，并转换为字符串
  • 未找到属性，则返回 undefined

如果属性名不是合法标识符（数字、包含空格标点）或需要通过运算得出的值，则必须使用方括号

调用表达式

写法：函数表达式后面接 ( arguments )

  • 如果有 return，则这个就是表达式值，如果没有，则返回 undefined
  • 括号前面是属性访问表达式，称为：方法调用，this指向调用对象

对象创建表达式

写法：new 函数调用表达式

  • 如果不传任何参数，括号可省略
  • 计算该表达式值时，首先创建空对象，将该空对象作为 this 调用函数，并将该对象返回
  • 如果构造函数内返回了一个对象，那么原来新创建的对象就被废弃了
