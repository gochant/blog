JavaScript 基础（二） 对象、数组

发表于 2014-10-07   |   分类于 Frontend

对象

对象的特征：

  • 包含属性
  • 原型
  • 类
  • 扩展标记

对象分类：

  • 内置对象：ES规范定义的对象或类
  • 宿主对象：JS的解释器所嵌入的宿主环境定义的
  • 自定义对象：JS代码创建的

属性分类：

  • 自有属性
  • 继承属性

属性的特性

  • 可写
  • 可枚举
  • 可配置

创建对象

  • 直接量
  • new
  • Object.create

原型

强类型、弱类型、无类型

属性访问和赋值

. 或 []

注意: ES3中.后不能是保留字，ES5放宽了限制

  • 查询不存在属性，返回 undefined
  • 对象不存在，查询属性抛 TypeError

属性赋值要么失败，要么赋值成功，要么创建新属性，属性赋值失败的情况：

  • 属性只读（但defineProperty有例外）
  • 属性是继承的，且只读
  • 对象不可扩展的

注意: ES3中，设置属性失败不报错，只是不改变值，ES5的严格模式中，会抛TypeError

删除属性

删除属性时，只是切断属性与宿主的联系，不会操作属性中的属性，因此在某些时候，容易引起内存泄露，严谨一些的做法可遍历属性对象，依次删除

delete 在删除成功、删除不存在属性和后面不是一个属性访问表达式时，均返回true；在删除可配置型为false的属性时，返回false

在严格模式中，删除不可配置属性会报TypeError

（通过变量声明和函数声明创建的全局对象属性是不可配置的）

非严格模式下，删除全局对象属性时，可省略全局对象，直接用属性，但在严格模式下不能这样做

检测属性

  • 检测自由和继承属性：”x” in o
  • 检测自有属性：o.hasOwnProperty(“x”)
  • 检测自有且可枚举的属性：o.propertyIsEnumerable(“x”)

枚举属性

  • for/in
  • Object.keys() 返回数组，获取所有可枚举的自有属性名称
  • Object.getOwnPropertyNames() 返回数组，获取所有自有属性名称

属性getter和setter

  • 存取器属性
  • 数据属性

存取器属性可创建只读、只写、可读写的属性

属性特性

数据属性特性有：value、writable、enumerable、configurable
存取器属性特性：get、set、enumerable、configurable

使用 Object.getOwnPropertyDescriptor() 获取某个对象属性的描述符

使用 Object.defineProperty() 和 Object.defineProperties() 设置属性，设置时不必包含所有属性，默认会设置为 false 或 undefined，并且不能修改继承的属性

设置属性有以下规则，违反会抛TypeError

  • 如果对象是不可扩展的，则不能新增属性，只能更新已有属性
  • 如果属性不可配置，则不能修改可配置型和可枚举性
  • 如果存取器属性不可配置，则也不能修改getter和setter，也不能转为数据属性
  • 如果数据属性不可配置，则不能转为存取器属性；不能将可写性从false改为true，但可从true改为false；如果它既不可配置又不可写，那么不能修改它的值，但如果它可配置但不可写，那可以修改它的值。

老式API

在老式浏览器中，可能存在 __lookupGetter 和 \lookupSetter、\defineSetter、\defineGetter__ 等方法

对象的属性

原型属性

获取

ES5：Object.getPrototypeOf()
ES3：o.constructor.prototype

检测

pro.isPrototypeOf(o)

类属性

是一段字符串，可通过 Object.prototype.toString 间接查看

可扩展性

所有内置对象和自定义对象都是显式可扩展

  • 检测是否可扩展

Object.isExtensible(o)

  • 设为不可扩展

Object.preventExtensions(o)

设为不可扩展后，不能转换回可扩展，不可扩展仅代表对象本身不可扩展，它原型链上的对象无法控制

  • 设为封闭

Object.seal

既不能扩展，所有属性也变为不可配置，可通过 Object.isSealed 检测是否是封闭的；设为封闭后，不能解封

  • 冻结对象

Object.freeze

设为不可扩展、不可配置，将所有数据属性设为只读（但具有setter的存取器属性不受影响），使用 Object.isFrozon 检测是否冻结

序列化对象

JSON.stringify
JSON.parse

  • NaN、Infinity、-Infinity被序列化为null
  • function 和 undefined 的属性都不会被序列化，RegExp 和 Error 会被序列化成空对象，日期对象会被序列化成 ISO 格式字符串

函数定义表达式

关于函数定义表达式，有几点注意：

  • 函数定义表达式一般用在更大的表达式中，例如赋值和调用；如果语句中只包含定义表达式，则可能会报错，因为它会被识别为函数声明，可以在外面加 ()，强制识别为表达式；
  • 命名的函数定义表达式的名称只能在函数内部使用，通过 toString 可以让外部看到；

注意：函数声明语句不是真正的语句（？），因此不能出现在条件、循环、try、with 语句中

函数调用

有四种调用方式：直接调用、作为方法调用、作为构造函数调用、间接调用

直接调用

严格模式，函数调用上下文为 undefined（不是全局对象），因此可用来判断是否是严格模式

作为方法

方法的链式调用

作为构造函数

如果构造函数显式使用return语句返回对象，那么调用表达式的值就是这个对象；如果使用了return，但没有返回值，或者返回原始值，则将会忽略返回值，使用新创建的对象

间接调用

使用 call 或 apply

函数的参数

实参、形参

arguments

  • 可接收不定实参（通过arguments[i]进行遍历）
  • 在ES5，arguments[x]不在与命名的形参共享值，同时严格模式中，不允许使用arguments作为标识符
  • arguments.callee 和 caller（非标准） 在 ES5 严格模式中禁止访问

最佳实践：

  • 如果不定长参数，一般将一个对象作为实参
  • 必要时候，检测传入实参的类型，可使用 || 进行默认值填充
  • 将函数作为值传递
  • 函数可设置属性值，这样该函数既可以调用，又可以当模块使用
  • 函数作为命名空间
  • 闭包的特性

函数的属性、方法和构造函数

length

代表函数定义是命名的形参数量，只读属性

arguments.length 则是实参个数

prototype

原型

call 和 apply 方法

  • ES5严格模式中，第一个参数永远为this
  • apply可以传入类数组，例如arguments
  • 传入函数，返回包装函数的函数称为泛函数，有时候叫做 monkey-patching

bind （ES5）

  • 使用ES3怎么模拟
  • ES5 bind 的一些隐藏特性
  • ES3 中无法模拟的部分

toString

返回函数定义字符串，内置函数返回 “[native code]”

Function() 构造函数

等价于创建一个匿名函数

  • 允许运行时创建并编译函数
  • 每次都会解析，创建函数，多次执行效率有问题
  • 在全局代码中执行
  • 实际中很少用到

可调用对象

看起来像函数，但实现上不是函数，这种对象会越来越少，例如老版本IE的window.alert、getElementById，以及浏览器的RegExp

尽量少使用
