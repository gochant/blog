内联

如果在XHTML文档里，应使用

外部脚本src

  • 如果指定了src，script标签内有内容，那这些内容被忽略，并且如果出现非注释，HTML5校验会报错

注意下面这种写法的意义

  • 当type为未知类型，则浏览器不执行里面的代码，这个特性用来保存模板等文本数据
  • 如果同时指定src和未知type，则不会请求脚本

url方式

  • 使用 javascript:void … 阻止返回内容，chrome 和 safari 默认忽略返回值
  • 制作书签：bookmarklet

JS程序的执行

  • 第一阶段：同步，文档载入执行阶段。载入文档内容，并执行 script标签的代码，基本上按照文档出现的顺序执行
  • 第二阶段：异步，事件驱动阶段

脚本的下载和执行都是同步阻塞的，这意味着，在它执行完成之前，HTML文档虽然已经载入，但不会被渲染成DOM树，可指定 defer 或 async 异步下载执行

关于async和defer的区别，见：http://segmentfault.com/q/1010000000640869 和 http://ued.ctrip.com/blog/?p=3121

线程模型

时间线

  1. 创建Document对象，Parse页面，document.readystate是 loading
  2. 遇到同步script元素，停止解析，加载执行脚本，该脚本可使用document.write，并访问部分dom树（它之前的元素）
  3. async脚本下载并执行（基本不能访问dom树，不能用document.write）
  4. 文档解析完毕后，readyState变成 interactive
  5. defer脚本执行，并能访问完整DOM树，不能使用document.write
  6. Document 触发 DOMContentLoaded 事件，进入异步事件驱动阶段
  7. 加载图片等内容，执行完异步脚本，readyState变成 complete，Window 上触发 load 事件

安全性

一些限制

  • 只有响应鼠标单击事件时才能弹出新窗口，否则会被拦截

相关研究：http://hi.barretlee.com/2014/09/07/onclick-in-javascript/

  • 可以关闭自己的窗口，但必须经过确认才能关闭其他窗口
  • file upload 元素的value是只读的

同源策略

这是对JS能够操作哪些Web内容的一条完整的安全限制。在使用iframe和打开新窗口生效

同源：即同协议、同主机、同端口

适用范围：

  • 绝大多数窗口和文档属性（脚本只能读取它所属文档来源相同的，但使用iframe可以访问父子窗口对象，但访问文档对象受限）
  • XMLHttpRequest请求

合理打破同源策略：

  • 子站点共享：改变 document.domain，可避免各子站点同源策略约束
  • 跨域资源共享（CORS）：使用 Origin 请求和Access-Control-Allow-Origin响应扩展HTTP，使Ajax请求规避同源策略
  • 跨文档消息：使用postMessage，实现安全通信

获取元素位置和尺寸

万能的 Element.getBoundingClientRect() 方法

该方法获取元素的位置坐标，该方法返回ClientRect对象，包括：top、left、right、bottom，还有 width、height（包括内边距和边框）

这里相对的坐标系是视口坐标系，即原点在视口左上角，为(0, 0)，可以为负数。

行内元素如果分为两行，则上面的方法获取到的是该元素的边界矩形，即将它们包裹住的一个矩形，如果要分别获取两行的矩形坐标，则使用 Element.getClientRects()，以上返回的是个静态数组，不是如NodeList那样是实时的。

offsetXXX 类属性

通过 Element.offsetWidth 和 Element.offsetHeight 获取元素的尺寸（包括内边距和边框）是最好的方案，虽然通过 getBoundingClientRect 能够获取到，但可能是个位数很大的小数（因为通过坐标算出来的）。

Element.offsetLeft 和 Element.offsetTop 可以获取元素的位置，但与上面有两点不同：

  • 该API获取到的是文档坐标系下的坐标
  • 如果该元素已定位，则该坐标是相对于定位祖先元素的坐标

clientXXX 类属性

Element.clientWidth 和 Element.clientHeight 与 offset*类方法相比，它们不包括边框，都不包括滚动条

clientTop 和 clientLeft 没啥用，指的是左（上）内边距上缘和边框上缘的距离，通常等于该地方的边框（除非有滚动条，但把滚动条放在左边、上边的浏览器，也太奇葩了）

scrollXXX 类属性

Element.scrollWidth 和 Element.scrollHeight 是元素的内容区域（clientXXX）加上元素溢出的内容部分
scrollLeft 和 scrollTop 则是获取或设置滚动条的位置

获取某位置有什么元素

使用 document 的 elementFromPoint：

该方法传入视口坐标，返回DOM树中嵌套最里面和z-index最上面的元素。如果该点在视口坐标外，则返回null

由于通常在事件处理函数中，事件对象已经包含了目标元素和坐标，因此该方法一般不常用。

CSS

设置内联样式

Element.style 它是一个 CSSStyleDeclaration 对象，它只返回显示设置在这个元素上的值，因此用作查询样式时用处不大

名字约定

  • 连字符大写，如：borderLeftWidth
  • 如果保留字加css前缀，如：cssFloat

设置属性时，必须要带上单位，直接赋 Number 是错误的

一种简单的设置内联样式的方式是直接设置style属性的字符串文本：

查询计算出的样式

window.getComputedStyle(element, null);

  • 该方法返回只读的 CSSStyleDeclaration
  • 样式都是绝对值，没有相对单位（em、%）或颜色（‘red’，都是rgb(a)）
  • 没有复合属性，因此没有 margin 只有 marginLeft
  • cssText 属性未定义

兼容性：IE8不支持，因此它使用 element.currentStyle，与之不同的是，该方法未将相对值转化为绝对值（也就是设置什么就是什么）

操作 CSS 类

Element.className

该属性是个字符串，因此多个类名不好操作，HTML5定义了：Element.classList，是个 DOMTokenList 对象，是实时的，定义了 add、remove、toggle、contains 等多个方法

兼容性：IE9及以下都不支持该属性

操作样式表

使用 document.styleSheets 获取一个 CSSStyleSheet 对象（简称：Sheet对象）的只读数组，包括 style 和 link 设置的样式表

启用禁用样式表

设置某个 Sheet 对象的 disabled 属性为 true 或 false，启用或禁用该样式表

设置规则

Sheet对象的 cssRules 属性，它是一个 CSSRule 数组，包含该样式表的所有规则，Rule 有一个 cssText 属性，可获取定义时的CSS文本

兼容性：IE8 不支持 cssRules，使用 rules 对象，与之等价，其他方法也有细微差异

Sheet 对象通过 insertRule、deleteRule 设置规则
