JavaScript 基础（四）事件、HTTP

发表于 2014-10-16   |   分类于 Frontend

事件处理

客户端JS采用了异步事件编程模型，简而言之，事件就是Web浏览器通知应用程序发生了什么事。事件的定义包括：

  • 事件类型（event type）
  • 事件目标（event target）
  • 事件对象（event object）
  • 事件传播（event propagation）

事件类型

事件类型来源：

  • DOM3 事件规范
  • HTML5 规范
  • 移动设备和触摸

分类：

  • 依赖于设备的输入事件

mousedown、mousemove、mouseup、keydown、keypress、keyup 等，touchmove、gesturechange 等

  • 独立于设备的输入事件

click、textinput（未广泛实现，） 等

  • UI事件

较高级的事件，包括：focus、change、submit 等

  • 状态变化事件

load、DOMContentLoaded；历史管理：popstate；离线API：online、offline；Ajax：readystatechange；本地文件异步IO：loadstart、 progress、loaded 等

  • 特定API事件

拖放：dragstart、dragenter、dragover、drop；播放器：waiting、playing、seeking、volumechange 等

  • 计时器和错误处理

表单事件

  • form 元素：submit、reset
  • 可点击的元素（如按钮、单选复选框）：click
  • 具有value的元素（文本框、选择框、复选框等）：change（对于文本输入域，只有离开元素时才会触发change）
  • 可得到焦点的元素：focus、blur

focus和blur不会冒泡，其他表单事件会，IE定义了不标准的focusin 和 focusout，支持冒泡，jQuery模拟了它，该事件正在标准化（TODO）

Window 事件

  • load
  • DOMContentLoaded、readystatechange
  • unload
  • beforeunload
  • onerror

img 也有 load、error，也有 abort（HTML5标准）

  • focus、blur：代表浏览器窗口得到或失去键盘焦点
  • resize、scroll

scroll 也能在可以滚动的元素上触发

鼠标事件

单击：mousedown -> mouseup -> click
拖动：mousedown -> mousemove (-> mouseup -> click)
双击：mousedown -> mouseup -> click -> mousedown -> mouseup -> click -> dblclick
鼠标悬停：mouseover、mouseout；mouseenter、mouseleave

由于mouseover和mouseout要冒泡，因此你无法检测鼠标是真的离开某个元素，还是仅仅从该元素的一个子元素进入到另一个子元素，使用mouseenter、mouseleave不冒泡版本会更方便；这个仅IE支持，DOM3 Event规范标准化了，jQuery模拟了它

鼠标滚轮：mousewheel，DOM3规范规定了更规范的wheel事件，取代mousewheel

在Firefox中，mousewheel 叫 DOMMouseScroll

键盘事件

keydown、keyup、keypress（产生可打印字符）

按下某个键：keydown -> keypress… -> keyup

新的DOM事件

新的DOM3 Event 规范的主要改进：

  • 使用冒泡的focusin、focusout 取代不冒泡的 focus、blur事件
  • 使用不冒泡的mouseenter、mouseleave 取代 mouseover、mouseout
  • 标准化 wheel
  • 使用 textinput，而不是 keypress，监视文本输入（包括：键盘、剪切粘贴、拖放等方式）

HTML5 事件

audio、video 事件：canplay、loadeddata
拖放事件：dragstart、drag
历史管理事件：Window：hashchange、popstate
表单验证事件：Form：invalid
离线Web：Window：online、offline
异步通信：message
异步IO：XMLHttpRequest、FileReader：loadstart、progress、loadend
Web存储：storage
打印：beforeprint、afterprint
……

触摸屏、移动设备事件

手势（两个手指的缩放旋转）：gesturestart、gestureend
低级触摸事件：touchstart、touchend、touchover
方位：orientationchanged

使用 addEventListener

使用对象：Window、Document、Element

addEventListener(事件名, 处理程序, false)

第三个参数代表是否注册为在捕获阶段的事件处理程序，除非特殊需求，应总是传递 false

removeEventListener(事件名, 处理程序, false)

IE9 之前不支持 addEventListener，使用 attachEvent 和 detachEvent，它与标准方法的区别是：

  • 没有第三个参数，因为IE事件模型不支持事件捕获
  • IE的第一个参数的事件名应该带 on
  • 未传入 event 参数
  • 事件处理程序内的this指向全局对象：Window
  • 可以为同一个事件注册多个相同的事件处理程序，注册了多少次，就调用多少次
  • 调用顺序是任意的，不是按照注册顺序调用

事件传播

三个阶段：

  • 事件捕获
  • 触发当前元素的事件处理程序
  • 事件冒泡

不冒泡的事件：focus、blur、scroll、mouseenter、mouseleave、load 会冒泡，但会在document上停止冒泡，只有整个文档加载完，才会触发 window 的 load

事件捕获阶段最先调用 window 的捕获处理程序，然后沿着DOM树向下，调用到目标元素的父级元素，不调用目标对象本身注册的处理程序；而事件冒泡刚好相反
事件冒泡可实现事件代理，而事件捕获则提供了事件到达目标前查看的机会，可用于程序调试、事件取消

事件取消

取消默认行为

标准用法 event.preventDefault()；在IE8中，可以设置event.returnValue 为 false来达到同样效果；在对象属性和HTML属性事件处理程序中，直接 return false

在标准中，Event对象定义了 defaultPrevented 查看是否阻止了默认行为

取消事件传播

标准用法 event.stopPropagation()，能阻止后续事件的传播，但无法阻止定义在同一对象的其他处理程序，它能工作在捕获阶段，事件目标本身和冒泡阶段

IE8的事件对象有一个cancelBubble属性，设置为true则能阻止事件的进一步传播

使用 event.stopImmediatePropagation() 阻止事件传播，也能阻止在同一对象上注册的其他事件调用

接收响应

request的几个属性：status 和 statusText 分别以数字和文本的方式返回HTTP状态码

使用方法 getResponseHeader() 和 getAllResponseHeaders() 查询响应头

使用 responseText 中获取响应主体文本，或者 responseXML 中获取 XML、XHTML、HTML 的 Document

使用 readyState 查询请求的状态

值 常量 含义
0 UNSENT open还未调用
1 OPENED open已调用
2 HEADERS_RECEIVED 接收到头信息
3 LOADING 接收到响应主体
4 DONE 响应完成

实际使用中，应监听 readystatechange 事件，获取 request.readyState 属性进行判断，可使用 on+xxx 监听，也可使用 addEventListenr，通常使用 onreadystatechange 更容易（因为通常只监听一次）

每次readyState变化，都会触发readystatechange事件，但有些例外：当状态变为0或1时，可能不会触发；调用send时，状态为改变（仍为1），也通常会触发；某些浏览器会多次触发3，以便进行进度反馈

同步的响应

如果是同步响应（open 的第三个参数传入false），则不用监听readystatechange事件，直接读取即可

responseText 只能处理文本，不能处理二进制数据，XHR2定义了处理二进制响应的方法，但浏览器厂商还未实现（使用Blob）

进度事件

XHR2定义了更多有用的事件集，因此可以不必使用readystatechange事件

当调用 send 时，触发loadstart事件，然后加载响应时，通常每隔50ms，触发progress事件（如果完成太快，可能就不会触发），完成后，触发load事件；如果无法完成，由三种情况：请求超时，则触发timeout，请求中止，触发abort，网络错误，会触发error；浏览器对于请求只会触发load、timeout、abort、error中的一个，XHR2规定，出现其中一个，则触发loadend事件
